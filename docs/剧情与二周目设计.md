剧情与二周目设计 (NG+ / Meta 架构)
1. 核心体验概念 (Meta 交互与第四面墙)
游戏的重玩不仅是重新开始，角色（尤其是阿斯特）与主角潜意识会意识到“重开”这一事实。通过继承不同存档周目的“记忆”（跨存档 profile 变量），改变剧情描述和角色对话。

A. 记忆继承与“既视感” (Deja Vu)
前世记忆：当玩家开启新周目时，如果满足条件（memory["played"] 或 memory["known_codes"] 等），文案将出现变化。
示例（scene_gate）：

python
if memory["played"]:
    text.config(text="你再次站在铁门前。\n这一次……你记得这里。")
else:
    text.config(text="你第一次来到庄园。")
记住密码：主角潜意识保留已解密的密码。如果上一周目玩家已经得出了密码 188，再次游玩相同场景时，UI 会多出特殊选项或直接提示。
示例（温室密码箱）：

python
if "188" in memory["known_codes"]:
    print("你突然想起……密码是188。")

状态变量：state["astre"]["stage"]（0 正常 → 1 怀疑 → 2 压迫 → 3 真相）

阶段更新逻辑（基于伏笔数量与仪式进度）：

def update_astre():
    f = len(state["foreshadow_seen"])
    r = state["world_state"]["ritual_progress"]
    if f >= 2: stage = 1
    if f >= 4: stage = 2
    if f >= 6 or r >= 2: stage = 3

2. 融入程序的开发细节
A. 状态层 (core/game-state.js 或 Python 全局 state)
使用 state["foreshadow_seen"] 记录所有触发的伏笔。

使用 state["world_state"]["ritual_progress"] 记录仪式进度。

使用 state["scene_states"] 记录每个场景的本地状态（如是否已调查某物）。

每次触发 add_foreshadow(key) 时自动更新阿斯特阶段。

B. 渲染层 (ui/scene-renderer.js 或 tkinter 文本组件)
提供“系统故障”或“文本篡改”的动画效果：检测到当前周目数大于一定值，或当前是打破第四面墙的场景时，给文本加上闪烁效果。

使用 creepy_text() 统一输出，在高阶段自动追加压迫描述。

C. 脚本层 (文本/*.txt 或场景定义)
支持条件判断语法，例如：

text
[条件: hasEnding('True_End')] → 显示不同描述
[效果: +flag:astre_suspicion] → 增加阿斯特警惕
场景描述可依赖 lambda state: ... 动态生成，根据 memory 或 foreshadow 改变文本。

3. 结局系统（全过程分支收束）
A. 核心改造思路
不再让玩家在最后选择结局，而是根据玩家一路上的行为自动判定结局。最终章只做揭示，不做决定。

B. 结局判定变量
python
ending_flags = {
    "fear": 0,      # 地下室：逃避 vs 面对
    "empathy": 0,   # 画室：理解 vs 忽略
    "logic": 0,     # 钟楼：推理 vs 跳过
    "obsession": 0  # 执着速通 / 收集
}
C. 各谜题对变量的影响
谜题	行为	影响
地下室	多次失败后放弃	fear -1
使用提示跳过	fear 0
坚持完成	fear +2
画室	跳过文本	empathy -2
正常阅读	empathy +1
解锁隐藏剧情	empathy +3
钟楼	直接试答案	logic 0
推理解谜	logic +2
解锁隐藏解法	logic +3
全局	速通且全收集	obsession +2
D. 最终结局映射

if obsession >= 2: ending = "囚徒"

elif empathy >= 3: ending = "讲述者"

elif fear >= 2: ending = "守护者"
else: ending = "无知"
