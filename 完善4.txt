## 彩蛋与 Meta 元素：完整实施方案

以下为三大 Meta 系统的详细技术实现，包括数据结构、对话系统、隐藏房间触发、日期彩蛋等。

---

## 一、阿斯特的“第四面墙”对话系统

### 1.1 设计目标

- 在多周目游戏中，阿斯特（管家/幽灵/画中声音）逐渐意识到玩家在“重玩”，台词从正常变为 Meta。
- 根据玩家的**轮回次数（playCount）**和**死亡次数（deathCount）**，动态选择不同层级的 Meta 台词。
- 台词可出现在：大厅管家对话、画室支线触发、最终密室、死亡结局等场景。

### 1.2 数据结构

在 `profile` 中已有 `play_count`，在 `memory.json`（全局记忆）中已有 `death_count`。我们直接在 `gameStore.js` 中增加计算属性或方法。

```javascript
// gameStore.js getters
metaLevel: (state) => {
  const playCount = state.profile.play_count;
  const deathCount = state.memory?.death_count || 0;
  if (playCount >= 3 && deathCount >= 5) return 'awakened';      // 彻底觉醒
  if (playCount >= 2 && deathCount >= 2) return 'aware';         // 有所察觉
  if (playCount >= 2) return 'suspicious';                       // 开始怀疑
  return 'normal';
}
```

## 二、隐藏的“开发者房间”

### 2.1 设计目标

- 玩家集齐全部成就（`谜语大师`）后，大厅出现一扇新门。
- 进入后展示：开发者留言、概念图、未使用的文本片段、制作人问答互动。
- 可加入简单的“感谢墙”或“彩蛋代码输入”。

### 2.2 数据结构

在 `profile` 中已有 `achievements` 列表，我们通过计算属性判断是否集齐所有成就。

```javascript
// gameStore.js getters
hasAllAchievements: (state) => {
  const totalAchievements = 25; // 根据你的成就总数调整
  return state.profile.achievements.length >= totalAchievements;
}
```

### 2.3 大厅动态添加“开发者房间”选项

修改 `hall_main` 的选项，根据 `hasAllAchievements` 动态添加：

```javascript
// hall.js
'hall_main': {
  // ... 原有描述和选项
  options: (store) => {
    const baseOptions = [
      { text: '仔细观察大厅壁炉的纸片', target: 'hall_fireplace' },
      // ... 其他原有选项
    ];
    if (store.hasAllAchievements && !store.getFlag('dev_room_unlocked')) {
      baseOptions.push({
        text: '✨ 一扇从未见过的门 ✨',
        target: 'dev_room_entry',
        effect: (store) => store.setFlag('dev_room_unlocked', true)
      });
    }
    return baseOptions;
  }
}
```

## 三、现实联动彩蛋

### 3.1 设计目标

- 在某个文本输入谜题（如壁炉密码、最终密室问题）中，输入特定的**现实日期**（如游戏发售日、制作人生日、历史事件）会触发隐藏对话或成就。
- 彩蛋可解锁特殊道具、成就，或直接显示一段幽默提示。

### 3.2 彩蛋配置

创建 `src/data/easterEggs.js`：

```javascript
export const easterEggs = [
  {
    id: 'release_date',
    keyword: '2024-10-31',  // 示例发售日

    message: '你输入的是谜语馆向公众开放的日子！那是你第一次来到这里。不，也许是你第一次“通关”的日子。',
    achievement: 'first_visitor',
    rewardItem: '时光怀表'

  {
```

### 3.3 修改文本输入谜题的验证逻辑

以 `hall_fireplace` 的凯撒密码谜题为例，原本只接受 `courtesy`。我们扩展验证器，同时检查彩蛋日期：

```javascript
// hall.js 中的 hall_fireplace 场景
input: {
  validate: (ans, store) => {
    const lowerAns = ans.trim().toLowerCase();
    // 正常谜底
    if (lowerAns === 'courtesy') return 'normal';
    // 检查彩蛋
    const egg = easterEggs.find(e => e.keyword === lowerAns);
    if (egg) return egg.id;
    return false;
  },
  onSuccess: (ans, store, result) => {
    if (result === 'normal') {
      // 正常跳转
      return 'musicroom_unlock';
    } else {
      // 彩蛋触发
      const egg = easterEggs.find(e => e.keyword === ans);
      store.showToast(`✨ 彩蛋触发：${egg.displayDate} ✨`);
      store.addClue(`你输入了特殊日期：${egg.displayDate}，${egg.message}`);
      if (egg.achievement) store.unlockAchievement(egg.achievement);
      if (egg.rewardItem) store.addItem(egg.rewardItem);
      // 彩蛋后仍允许正常继续（或直接跳转）
      return 'musicroom_unlock';
    }
  },
  failMsg: '密码错误，暗格发出一声沉闷的声响。',
  hints: [ ... ]
}
```

### 3.4 在多个谜题中嵌入彩蛋

可以在其他文本输入谜题（如卧室的“主人问题”、图书馆的密码锁等）也加入类似的彩蛋检测。为了复用，可以封装一个通用函数：

```javascript
// utils/easterEggHelper.js
import { easterEggs } from '@/data/easterEggs';

export function checkEasterEgg(input, store) {
  const normalized = input.trim().toLowerCase();
  const egg = easterEggs.find(e => e.keyword === normalized);
  if (egg) {
    store.showToast(`✨ 彩蛋触发：${egg.displayDate} ✨`);
    store.addClue(`你输入了特殊日期：${egg.displayDate}，${egg.message}`);
    if (egg.achievement) store.unlockAchievement(egg.achievement);
    if (egg.rewardItem) store.addItem(egg.rewardItem);
    return true;
  }
  return false;
}
```

然后在每个输入谜题中调用：

```javascript
validate: (ans, store) => {
  if (checkEasterEgg(ans, store)) return 'easter_egg_success';
  // 正常验证逻辑
  return ans === 'correct';
}
```
