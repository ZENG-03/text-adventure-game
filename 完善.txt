以下为“角色情感线强化”三大系统的具体技术实现方案，包括代码示例、设计思路和剧情示例，可直接集成到现有 Vue + Pinia 架构中。

## 一、“回忆碎片”系统

### 1.1 设计目标

- 在关键场景（画室完成、音乐室演奏后、支线关键节点）自动触发一段**第一人称闪回**。
- 闪回以**全屏半透明遮罩 + 手写体文字 + 淡入淡出**的形式呈现，模拟记忆涌上心头的效果。
- 每个闪回只触发一次（记录在 profile 中）。

### 1.2 数据结构

在 `gameStore.js` 的 `profile` 中添加：

```javascript
profile: {
  // ... existing
  triggered_flashbacks: [], // 已触发的闪回ID列表
  character_affection: {    // 角色好感度/共鸣值（后续使用）
    butler: 0,
    elenor: 0,
    elena: 0,
    thomas: 0
  }
}
```

### 1.4 Store 中增加触发方法

在 `gameStore.js` 的 `actions` 中添加：

```javascript
// 触发闪回（由场景调用）
async triggerFlashback(flashbackId) {
  if (this.profile.triggered_flashbacks.includes(flashbackId)) return;
  const flash = flashbacks[flashbackId];
  if (!flash) return;

  // 检查触发条件
  let conditionMet = false;
  const trigger = flash.trigger;
  if (trigger.type === 'flag') {
    conditionMet = this.getFlag(trigger.key) === trigger.value;
  } else if (trigger.type === 'playCount') {
    conditionMet = this.profile.play_count >= trigger.min;
  } else if (trigger.type === 'item') {
    conditionMet = this.run.items.includes(trigger.item);
  }

  if (!conditionMet) return;

  // 标记已触发
  this.profile.triggered_flashbacks.push(flashbackId);
  this.saveState();

  // 通过事件总线发送闪回事件
  emitter.emit('show-flashback', flash);
}
```

### 1.5 在场景中触发闪回

修改各房间的成功场景，在 `on_enter` 中调用触发方法。例如在 `studio.js` 的 `studio_gem_correct` 中：

```javascript
  // ...
```

## 二、角色好感度/共鸣值系统

### 2.1 设计目标

- 玩家在支线对话、关键选择中的态度，影响各角色的“共鸣值”。
- 共鸣值影响最终结局中的细节（角色是否微笑、是否赠送特殊道具、对话变化）。
- 不强制影响结局走向，但增强“我的选择有意义”的体验。

### 2.2 数据结构

已在 `profile` 中预留 `character_affection`，初始值均为 0，范围 -5 到 +10。

```javascript
character_affection: {
  butler: 0,   // 管家奥尔德斯
  elenor: 0,   // 埃莉诺（制琴师）
  elena: 0,    // 伊莲娜（画中女子）
  thomas: 0    // 托马斯（地质学家）
}
```

### 2.3 修改选项效果

在支线场景中，为不同选项添加 `effect` 来改变好感度。例如在 `butler.js` 的 `side_reveal_1` 中：

```javascript
// 原选项
{ text: '质问他的身份', target: 'side_reveal_1' }

// 修改为带效果
{ 
  text: '质问他的身份', 
  target: 'side_reveal_1',
  effect: (store) => {
    store.modifyAffection('butler', -2);
    store.addClue('管家对质问感到愧疚');
  }
},
{ 
  text: '平静地询问', 
  target: 'side_reveal_1_gentle',
  effect: (store) => {
    store.modifyAffection('butler', +3);
    store.addClue('管家愿意敞开心扉');
  }
}
```

### 2.4 Store 中增加好感度修改方法

```javascript
// gameStore.js actions
modifyAffection(character, delta) {
  const current = this.profile.character_affection[character] || 0;
  let newVal = current + delta;
  // 限制范围 -5 到 10
  newVal = Math.min(10, Math.max(-5, newVal));
  this.profile.character_affection[character] = newVal;
  this.saveState();
  
  // 可选：触发提示
  if (delta > 0) {
    this.showToast(`你与${this.getCharacterName(character)}的关系更加亲近了。`);
  } else if (delta < 0) {
    this.showToast(`你与${this.getCharacterName(character)}的关系变得疏远。`);
  }
},

getCharacterName(character) {
  const names = {
    butler: '管家奥尔德斯',
    elenor: '埃莉诺',
    elena: '伊莲娜',
    thomas: '托马斯'
  };
  return names[character] || character;
}
```

### 2.5 结局中根据好感度改变细节

在 `endings.js` 的真结局中，动态修改描述：


## 三、多结局的因果回溯（记忆之书）

### 3.1 设计目标

- 在游戏结局后，提供一个“记忆之书”界面。
- 以时间线或节点树形式，展示玩家在关键节点的选择，并解释如何导向当前结局。
- 允许玩家从某个节点重新开始（分支跳转），增加重玩价值。

### 3.2 数据结构

在 `profile` 中记录关键选择：

```javascript
profile: {
  // ...
  key_choices: []  // 存储 { nodeId, choiceText, timestamp, sceneId }
}
```

### 3.3 记录关键选择

在场景选项中，为关键选择添加 `recordChoice` 效果：

```javascript
{
  text: '选择成为传播者',
  target: 'side_ending_spreader',
  effect: (store) => {
    store.recordKeyChoice('final_choice', '成为传播者', 'final_chamber');
  }
}
```

在 `gameStore.js` 中添加方法：

```javascript
recordKeyChoice(nodeId, choiceText, sceneId) {
  this.profile.key_choices.push({
    nodeId,
    choiceText,
    sceneId,
    timestamp: Date.now()
  });
  this.saveState();
}
```

### 3.4 记忆之书组件（BookOfMemories.vue）

```vue
<template>
  <div class="memory-book-overlay" @click.self="close">
    <div class="memory-book">
      <h2>记忆之书</h2>
      <p class="subtitle">你走过的路，铸就了你的结局</p>
      
      <div class="timeline">
        <div 
          v-for="(choice, index) in keyChoices" 
          :key="index"
          class="timeline-node"
        >
          <div class="node-marker"></div>
          <div class="node-content">
            <div class="node-scene">{{ getSceneName(choice.sceneId) }}</div>
            <div class="node-choice">{{ choice.choiceText }}</div>
          </div>
        </div>
      </div>

        <h3>结局分析</h3>
        <p>{{ endingExplanation }}</p>
      </div>
      
      <div class="buttons">
        <button class="restart-btn" @click="restartFromBeginning">重新开始</button>
        <button class="close-btn" @click="close">关闭</button>
      </div>
    </div>
  </div>

<script setup>
import { computed } from 'vue';
import { useGameStore } from '@/store/gameStore';

const store = useGameStore();
const emit = defineEmits(['close']);

const keyChoices = computed(() => store.profile.key_choices || []);

const getSceneName = (sceneId) => {
  const map = {
    'final_chamber': '最终密室',
    'side_butler_quarters': '管家起居室',
    'side_painting_trigger': '画室',
    // ...
  };
  return map[sceneId] || sceneId;
};

const endingExplanation = computed(() => {
  // 根据玩家的关键选择和好感度生成动态分析
  const choices = keyChoices.value;
  const aff = store.profile.character_affection;
  let text = '你的旅程中，';
  if (choices.some(c => c.choiceText === '成为传播者')) {
    text += '你选择了将谜语公之于众，这让你获得了自由，但也失去了庄园的物质财富。';
  } else if (choices.some(c => c.choiceText === '成为守护者')) {
    text += '你选择了承担守护的责任，虽然失去了自由，但获得了内心的安宁。';
  }
  if (aff.elena >= 5) {
    text += ' 伊莲娜感受到了你的理解，她的灵魂得以安息。';
  }
  return text;
});

const restartFromBeginning = () => {
  // 重置游戏
  store.startNewGame();
  emit('close');
  window.location.reload(); // 或路由跳转
};

const close = () => {
  emit('close');
};
</script>
```
