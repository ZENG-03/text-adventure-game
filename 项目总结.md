# 文字冒险-解谜 项目总结

更新日期：2026-04-07

## 1. 项目定位

这是一个以“庄园探索 + 多房间谜题 + 多结局”为核心的文字冒险解谜项目，包含：

- Python 终端版本（运行主流程、状态管理、存档读档）
- Web/JavaScript 版本（在浏览器中展示场景与交互）
- 文本策划与自动构建工具链（由文本稿生成 JS 场景数据）

## 2. 主要模块结构

### Python 运行核心

- `main.py`：程序入口，加载场景并启动引擎
- `engine.py`：包含 `GameState`、`Scene`、`Option`、`GameEngine`
	- 支持道具/线索/徽章/分支状态
	- 支持 `save`、`load`、`inv`、`q` 指令
- `data.py`：构建 Python 版场景字典（主线 + 谜题 + 房间）
- `state_store.py`：Python 侧存档/状态持久化读写

### Web 运行核心

- `index.html`：前端页面入口
- `js/game-engine.js`：浏览器端状态与场景驱动逻辑
- `js/game-scenes.js`：场景数据（手写 + 自动生成区块）
- `js/state-store.js`：Web 侧状态存储与读写同步

### 构建与处理脚本

- `build_js.py`、`build_js_all.py`：将 `文本/*.txt` 解析并写入 `js/game-scenes.js`
- `replace_placeholder_scenes.py`、`replace_enhanced.py`：场景替换/增强处理
- `analyze_scenes.py`：场景分析辅助脚本
- `validate_scenes.py`：场景结构、目标链接与资源路径校验

### 策划文本与数据

- `文本/`：主线、支线、各房间谜题、结局、建议文档
- `data/scenes.dsl.json`：结构化场景数据文件

## 3. 当前玩法与机制要点

- 以大厅为枢纽，串联图书馆、音乐室、温室、画室、地下室、钟楼等谜题空间
- 通过解谜获得徽章与关键道具，逐步解锁更深层区域
- 支持条件分支（是否持有道具、是否具备线索、状态标记）
- 累积徽章数量达到阈值后可触发最终密室与结局线
